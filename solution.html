User Flow

From Setup to
First Savings

A FinOps engineer or cloud admin can get CloudCash running and seeing their first automated savings in under 10 minutes.

1

Connect AWS Account

User connects their AWS account via IAM role with read + limited write permissions. CloudCash requests only what it needs — least privilege by design.

IAM Role Setup AWS Config ~2 minutes
2

AI Scans Your Infrastructure

Amazon Bedrock ingests 90 days of Cost Explorer data, pulls CloudWatch utilization metrics for every resource, and runs Compute Optimizer recommendations. The AI produces a prioritized list of waste.

Amazon Bedrock Cost Explorer CloudWatch ~3 minutes
3

Review Recommendations Dashboard

User sees a ranked list of opportunities — idle EC2 instances, over-provisioned RDS databases, recommended Reserved Instances — each with estimated savings and risk rating.

Interactive Dashboard Savings Estimates Risk Ratings
4

Set Automation Rules

User configures which action categories CloudCash can auto-execute vs. which require approval. Example: "Auto-stop EC2 idle > 7 days, ask me before purchasing Reserved Instances."

Approval Thresholds AWS Lambda Config Policy Builder
5

💰 Autopilot Saves Money Continuously

CloudCash runs every 15 minutes via EventBridge. Every action is logged to DynamoDB. Monthly savings reports are generated automatically in plain English by Amazon Bedrock and emailed to stakeholders.

EventBridge Scheduler DynamoDB Logs Bedrock Reports Email Alerts
